SEARCH OPTIONS
==============

When you perform a search, you'll now be asked:

1. "Show full text? (y/n, default y):"
   - y (default): Shows complete chunk text
   - n: Shows 500-character preview

2. "Save results to file? (y/n, default n):"
   - y: Saves full results to data/search_results_TIMESTAMP.txt
   - n (default): Only shows in console

1. USE CONTEXT SEARCH [T] for comprehensive information
   - Shows full sections, not just isolated chunks
   - Better for understanding workflows and procedures

2. INCREASE context window for complex topics

4. INCREASE number of results (k) when:
   - Query is broad (e.g., "authentication")

5. USE HYBRID SEARCH [H] as default
   - Best accuracy (combines semantic + keyword)
   - Works well for technical queries
   - Finds both conceptual and exact matches

UNDERSTANDING SCORES
====================

Each result shows three scores:

1. Semantic Score (0.0 - 1.0)
   - Measures conceptual similarity
   - Based on vector embeddings
   - Higher = more semantically related

2. Keyword Score (0.0 - 1.0)
   - Measures exact term matching
   - Based on BM25 algorithm
   - Higher = more keyword matches

3. Hybrid Score (0.0 - 1.0)
   - Combined score: 0.7 * semantic + 0.3 * keyword
   - Best indicator of overall relevance
   - Results are sorted by this score

INTERPRETING SCORES:

  Score     Quality     Interpretation
  ─────────────────────────────────────────────────
  0.9-1.0   Excellent   Highly relevant, exact match
  0.7-0.9   Good        Relevant, on-topic
  0.5-0.7   Fair        Somewhat relevant
  0.3-0.5   Poor        Loosely related
  0.0-0.3   Very Poor   Not relevant

USING RESULTS WITH AI AGENTS
=============================

The search results are designed to be agent-friendly:

PYTHON EXAMPLE:

from src.vector_db.search_engine import HybridSearchEngine

# Initialize
search = HybridSearchEngine()

# Get results
results = search.search(
    query="how to configure fingerprinting",
    k=10,
    search_mode='hybrid'
)

# Extract all matched text
all_context = "\n\n".join([
    f"[From {r.chunk_metadata.doc_name}, Page {r.chunk_metadata.page_number}]\n"
    f"{r.chunk_metadata.text}"
    for r in results
])

# Pass to your LLM
prompt = f"""
Based on the following documentation:

{all_context}

Generate detailed test cases for the fingerprinting feature.
"""

# Use with your agent framework
test_cases = your_llm.generate(prompt)

TROUBLESHOOTING OUTPUT ISSUES
==============================

Q: Results are still truncated?
A: Make sure you answered 'y' to "Show full text?" prompt

Q: No results found?
A: - Check [V] stats to verify docs are ingested
   - Try [H] hybrid search instead of [S] semantic
   - Increase number of results (k=10 or k=20)

Q: Results don't seem relevant?
A: - Use more specific query terms
   - Try [K] keyword search for exact terms
   - Check hybrid scores (should be >0.5 for relevance)

Q: Want even more context?
A: - Use [T] context search
   - Increase context window (2-5 chunks)
   - Increase number of results

Q: File not saving?
A: - Check data/ directory exists and is writable
   - Check logs: data/logs/rag_system.log
